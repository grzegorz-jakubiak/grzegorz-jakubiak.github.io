Runno

Feb 5, 2022
1 min read
Categories: Ruby

It has been a while, and I have something special for y’all. Through this week’s issue of Ruby Weekly (#589) I discovered runno. It is an in-browser code runner. It uses WebAssembly and supports multiple runtimes. Ruby is one of them, and that’s what got me interested…

Talk is cheap. Show me the code.

– Linus Torvalds

Ruby has added official support for WASI based WebAssembly. This has prompted the maintainer of runno to add support for Ruby to his code runner.

runno has a web component responsible that you can add to your website, and it does the magic.

puts "Hello world!"


We all know “Hello world”, let’s see what else it can do:

numbers = [1, 2, 3, 4] numbers.each { print "#{_1 ** 2} " }


As we can see we can run any ruby code. Well… not exactly any. There are some limitations mentioned in the Ruby pull request.

I found out that runno doesn’t really work in Firefox, and I created an issue on github. I also noticed the web component behaves weird sometimes, and doesn’t show the full terminal with the output.

All in all, it’s a great tool that I want to run code samples.

Take care!
